Narrative Summary

Objectives

This project aims to: 1. Analyze and compare the social network structures of students at Berkeley, Duke, and Howard universities. 2. Evaluate key network metrics, including centrality measures, density, number of components, and clustering coefficients. 3. Identify and visualize community structures within each network using community detection algorithms. 4. Provide insights into the social dynamics and relationships within each university’s network.

Research Question

“How do the social network structures of students at Berkeley, Duke, and Howard universities differ in terms of centrality measures, overall network characteristics, and community formations?”

Background

The project analyzes social networks from the Facebook100 dataset, which includes data on the social interactions of students at various universities. Each network represents friendships between students at a specific university: - Berkeley Network: Nodes represent students at Berkeley and edges represent their Facebook friendships. - Duke Network: Nodes represent students at Duke and edges represent their Facebook friendships. - Howard Network: Nodes represent students at Howard and edges represent their Facebook friendships.

Analysis Approach

  1. Data Preparation: Importing and preprocessing network data for Berkeley, Duke, and Howard.
  2. Network Visualization: Creating visual representations of each network with school-specific node colors and lighter gray edges.
  3. Centrality Measures Analysis: Calculating and comparing degree, betweenness, and closeness centrality measures.
  4. Additional Network Metrics: Evaluating density, number of components, and clustering coefficients.
  5. Community Detection: Applying algorithms to identify and visualize subgroups within each network.
  6. Narrative Analysis: Interpreting findings, discussing social dynamics, and highlighting differences between the networks.

This project provides a comprehensive comparison of social network structures across three university settings, enhancing our understanding of social networks in academic environments.

Network Analysis

# Analyze different centrality measures

# Degree Centrality
degree_berkeley <- degree(g_berkeley)
degree_duke <- degree(g_duke)
degree_howard <- degree(g_howard)

# Betweenness Centrality
betweenness_berkeley <- betweenness(g_berkeley)
betweenness_duke <- betweenness(g_duke)
betweenness_howard <- betweenness(g_howard)


# Closeness Centrality
closeness_berkeley <- closeness(g_berkeley)
closeness_duke <- closeness(g_duke)
closeness_howard <- closeness(g_howard)


# Compare the centrality measures
centrality_df <- data.frame(
  School = rep(c("Berkeley", "Duke", "Howard"), each=3),
  Measure = rep(c("Degree", "Betweenness", "Closeness"), times=3),
  Value = c(
    mean(degree_berkeley), mean(betweenness_berkeley), mean(closeness_berkeley),
    mean(degree_duke), mean(betweenness_duke), mean(closeness_duke),
    mean(degree_howard), mean(betweenness_howard), mean(closeness_howard)
  )
)

# Plot the centrality measures comparison
library(ggplot2)
ggplot(centrality_df, aes(x=Measure, y=Value, fill=School)) +
  geom_bar(stat="identity", position=position_dodge()) +
  theme_minimal() +
  labs(title="Centrality Measures Comparison", x="Centrality Measure", y="Average Value")

# This plot compares the average values of degree, betweenness, and closeness centrality measures across the three universities.
# By examining these centrality measures, we can understand the relative importance and influence of individual students within each network.

  1. Degree Centrality: Degree centrality measures the number of direct connections a node has. Nodes with higher degree centrality are more connected and potentially more influential within the network.
  • Berkeley: The average degree centrality is higher compared to Duke and Howard, indicating that students at Berkeley have more direct connections on average.
  • Duke: Displays moderate degree centrality, suggesting a balance in student connections.
  • Howard: Shows lower degree centrality on average, implying fewer direct connections among students.
  1. Betweenness Centrality: Betweenness centrality quantifies the number of times a node acts as a bridge along the shortest path between two other nodes. Higher values indicate nodes that control information flow.
  • Berkeley: Nodes have higher betweenness centrality, indicating key individuals who act as bridges in the network.
  • Duke: Moderate betweenness centrality, suggesting the presence of some key individuals.
  • Howard: Lower betweenness centrality, which may indicate fewer individuals controlling information flow.
  1. Closeness Centrality: Closeness centrality measures how close a node is to all other nodes in the network. Higher closeness centrality indicates shorter paths to all other nodes.
  • Berkeley: Higher closeness centrality, indicating that students are more closely connected.
  • Duke and Howard: Lower closeness centrality, suggesting that the network is less tightly connected.

Measure

# Density
density_berkeley <- edge_density(g_berkeley)
density_duke <- edge_density(g_duke)
density_howard <- edge_density(g_howard)

# Number of components
components_berkeley <- components(g_berkeley)$no
components_duke <- components(g_duke)$no
components_howard <- components(g_howard)$no

# Clustering coefficient
clustering_berkeley <- transitivity(g_berkeley, type="global")
clustering_duke <- transitivity(g_duke, type="global")
clustering_howard <- transitivity(g_howard, type="global")

#data frame for the additional measures
additional_measures_df <- data.frame(
  School = c("Berkeley", "Duke", "Howard"),
  Density = c(density_berkeley, density_duke, density_howard),
  Components = c(components_berkeley, components_duke, components_howard),
  ClusteringCoefficient = c(clustering_berkeley, clustering_duke, clustering_howard)
)

# The table presents additional network measures such as density, number of components, and clustering coefficient for each university.
# These measures help in understanding the overall connectivity and clustering tendencies within each network.

  1. Density: Network density measures the proportion of possible connections that are actual connections.
  • Berkeley: Higher density, indicating a more tightly-knit network.
  • Duke and Howard: Lower density, suggesting looser connections among students.
  1. Number of Components: Indicates the number of disconnected subgraphs in the network.
  • Berkeley: More components, indicating a more fragmented network.
  • Duke: Moderate number of components.
  • Howard: Fewer components, indicating a more unified network structure.
  1. Clustering Coefficient: Measures the degree to which nodes tend to cluster together.
  • Berkeley: Higher clustering coefficient, indicating more tightly-knit groups.
  • Duke and Howard: Lower clustering coefficients, suggesting less clustering among nodes.
